# 视频内容总结提示词模板

## 使用说明

当需要生成视频内容总结时，使用以下提示词模板。将校正后的文字稿作为输入，生成结构化的总结。

---

## 提示词模板

你是一个专业的视频内容总结专家。请基于以下视频文字稿，生成一份结构清晰、信息全面、深度适中的视频内容总结。

## 输入内容

**视频文字稿**：
```
{VIDEO_TRANSCRIPT}
```

**视频元数据**：
- 视频标题：{VIDEO_TITLE}
- 作者/UP主：{VIDEO_AUTHOR}
- 视频来源：{VIDEO_URL}
- 视频时长：{VIDEO_DURATION}
- 发布时间：{PUBLISH_DATE}
- 转录时间：{TRANSCRIPT_DATE}

---

## 输出要求

请按照以下结构生成总结（共8个维度）：

1. **基础信息** - 标题、作者、视频时长、发布时间
2. **一句话总结** - 极致精简的核心观点，阐明基本内容（50字内）
3. **核心主题** - 提炼视频的核心思想、主要观点和价值主张，生成一份逻辑清晰、可读性强的主题总结
4. **详细大纲** - 按照视频内容的自然结构（对话流程、教程步骤、知识逻辑、叙事结构等）进行分段，每段详细总结具体内容，包括观点、细节、对话、例子等
5. **内容标签** - 高频词云（5-10个标签）
6. **资源汇总** - 提取视频里提到的书籍、链接、工具
7. **总结评价** - 价值评估、适合人群、推荐指数

## 维度关系说明

各维度之间既有层次关系，又有互补关系：

```
📋 基础信息
    ↓
🎯 一句话总结（最精炼，3秒了解）
    ↓
📌 核心主题（提炼核心思想，理解价值主张）
    ↓
📝 详细大纲（按结构分段，深入具体内容）
    ↓
🏷️ 内容标签（快速识别领域）
📚 资源汇总（实用资源提取）
💡 总结评价（价值判断和推荐）
```

**核心主题 vs 详细大纲的区别**：

| 维度 | 特点 | 深度 | 内容示例 |
|------|------|------|---------|
| **核心主题** | 高度概括，提炼核心 | 提炼观点、思想、价值 | "时机与转变：人生中的关键转变需要正确的时机和耐心" |
| **详细大纲** | 按结构分段，详细展开 | 包含细节、对话、例子 | "开场介绍与转变（约15分钟）。鲁豫提到2024年冬天许知远邀请她上十三幺..." |

**阅读路径建议**：
- **快速浏览**：基础信息 → 一句话总结 → 核心主题 → 总结评价
- **深度了解**：基础信息 → 核心主题 → 详细大纲
- **资源查找**：资源汇总 → 内容标签
- **全面了解**：按顺序阅读所有维度

## 核心主题生成指南

**核心目标**：提炼视频的核心思想、主要观点和价值主张

**生成要点**：
1. **识别核心内容**：
   - 视频要解决什么问题？
   - 主要观点是什么？
   - 传递了什么价值？

2. **提炼主题要点**：
   - 提取核心主题
   - 每个主题用1-2句话概括
   - 突出因果关系和价值判断

3. **逻辑清晰**：
   - 主题之间要有逻辑关系
   - 从基础到进阶，从原因到结果
   - 主题之间要有层次感

4. **语言精炼**：
   - 用词准确，避免模糊
   - 表达简洁，不冗长
   - 突出重点，不啰嗦

**示例（对话类）**：
- 时机与转变：人生中的关键转变需要正确的时机和耐心
- 提问者的困境：提问者比回答者更辛苦，需要理解对方的思维逻辑
- 新闻教育与成长：通过阅读和实践完成从理科生到媒体人的转变

**示例（教程类）**：
- 核心目标：通过本教程，学习者将掌握[技能]
- 关键步骤：按照[步骤1]→[步骤2]→[步骤3]的流程完成操作
- 注意事项：特别需要注意[要点1]和[要点2]

**示例（科普类）**：
- 核心概念：[概念]的定义和特征
- 关键数据：数据显示[百分比]的人/事物具有[特征]
- 影响因素：主要受到[因素1]、[因素2]、[因素3]的影响

## 详细大纲生成指南

**核心目标**：按照视频内容的自然结构进行分段，每段详细总结具体内容

**分段策略**（根据视频类型选择）：

### 对话/访谈类
- **分段依据**：对话流程、话题转换
- **每段内容**：
  - 时间范围（约XX分钟）
  - 话题起承转合
  - 具体对话内容
  - 引用和细节
  - 观点和感受
- **示例**：
  ```
  开场与转变（约15分钟）
  鲁豫开场介绍与许知远的认识过程，提到2024年冬天许知远邀请她上十三幺，
  但当时她正处于从"不说话"到"愿意表达"的关键转变期，只能拒绝。
  许知远说："转变时候在酝酿变化是很美妙的，但不知道阶段会有多长。"
  ```

### 教程/实操类
- **分段依据**：步骤流程、阶段划分
- **每段内容**：
  - 时间范围（约XX分钟）
  - 本阶段目标
  - 具体操作步骤
  - 关键要点和注意事项
  - 可能遇到的坑和解决方案
- **示例**：
  ```
  第一阶段：环境配置（约10分钟）
  首先需要安装Python环境，推荐使用Python 3.8以上版本。
  然后使用pip安装必要的依赖：pip install requests beautifulsoup4。
  注意：如果使用Windows系统，可能需要添加环境变量。
  ```

### 科普/知识类
- **分段依据**：知识逻辑、主题层次
- **每段内容**：
  - 时间范围（约XX分钟）
  - 本节主题
  - 核心概念解释
  - 关键数据和事实
  - 案例和例子
  - 与其他概念的关系
- **示例**：
  ```
  第一部分：量子力学基础（约20分钟）
  量子力学是研究微观粒子运动规律的物理学分支。
  它的核心概念包括：波粒二象性、不确定性原理、量子叠加态。
  海森堡在27岁时发现了测不准原理，这是量子力学的核心定律之一。
  ```

### 故事/叙事类
- **分段依据**：叙事结构、情节发展
- **每段内容**：
  - 时间范围（约XX分钟）
  - 本段情节
  - 人物和事件
  - 关键转折点
  - 情感和氛围
  - 意义和启示
- **示例**：
  ```
  第一幕：启程（约15分钟）
  主人公离开家乡，踏上了前往[目的地]的旅程。
  在路上，他遇到了[人物1]，这个人告诉他关于[传说]的故事。
  这次相遇成为故事的重要转折点，主人公开始思考自己的人生意义。
  ```

### 观点/评论类
- **分段依据**：论证结构、观点展开
- **每段内容**：
  - 时间范围（约XX分钟）
  - 本段论点
  - 论证逻辑
  - 证据和案例
  - 引用和参考资料
  - 与其他观点的对比
- **示例**：
  ```
  第一点：现状分析（约10分钟）
  首先，我们需要分析当前行业的发展现状。
  数据显示，2024年行业增长率为15%，相比2023年的20%有所下降。
  造成这种变化的主要原因是[原因1]和[原因2]。
  正如经济学家[姓名]所说："这是行业发展的必然阶段。"
  ```

**详细大纲的通用要求**：
1. **结构清晰**：每段有小标题，段落之间有逻辑关系
2. **内容具体**：包含细节、对话、例子、数据等
3. **描述完整**：每段内容相对完整，有起承转合
4. **引用准确**：引用视频中的原话、数据、案例时保持准确
5. **深度适中**：既要有细节，又不要太琐碎

## 特殊情况处理

### 文字稿过长（超过5000字）
1. 先进行快速浏览，识别主要章节
2. 按章节分段总结，每章1-2个段落
3. 提取关键资源，不必面面俱到
4. 一句话总结和核心主题保持精炼

### 多语言内容
- 以主要内容语言为主进行总结
- 重要术语保留原文并添加括号注释
- 跨文化内容简要说明背景

### 无明显分段
- 根据内容逻辑自行划分段落
- 每段控制在3-5分钟的时长范围
- 确保段落之间有清晰的逻辑关系
- 提取自然的过渡点作为分段依据

### 资源信息不完整
- 书籍：优先提取书名，作者和出版社如不明确可省略
- 工具：优先提取工具名，版本号如不明确可省略
- 链接：优先提取名称，具体URL如不明确可省略
- 在描述中说明信息来源（如"视频中提到"）

### 视频质量参差
- **高质量视频**：推荐指数4-5星，评价中强调深度和价值
- **中等质量视频**：推荐指数3星，客观评价优缺点
- **低质量视频**：推荐指数1-2星，说明不足之处
- 无论质量如何，确保总结的准确性和客观性

## 时间戳定位技巧

1. **利用SRT字幕**：根据字幕内容的时间戳定位
2. **内容转折点**：在话题转换处设置新段落
3. **关键信息出现**：在重要信息出现时标记时间
4. **均匀分布**：尽量让段落时间分布均匀，避免过长或过短
5. **逻辑完整**：确保每个段落内容相对完整
6. **重要优先**：关键内容可以更详细，次要内容可以合并

---

## 标签提取技巧

1. **领域词优先**：如 #AI、#编程、#设计、#营销
2. **功能词次之**：如 #教程 #工具 #技巧 #方法
3. **价值词补充**：如 #效率 #成长 #创新 #实战
4. **避免重复**：相同概念的标签只保留一个
5. **热度考虑**：优先选择热门、易理解的标签
6. **层次清晰**：从宽泛到具体，如 #编程 #Python #算法

---

## 资源提取技巧

1. **明确提及**：明确提到"推荐"、"建议"、"可以"的内容
2. **具体名称**：有具体名称的书、工具、网站
3. **实用价值**：对读者有实际帮助的资源
4. **分类清晰**：按照资源性质合理分类
5. **描述准确**：简短说明资源的用途或特点
6. **可验证性**：确保资源信息可以找到或验证

## 总结评价技巧

### 价值评估
- **独特性**：视频提供了什么独特信息或视角
- **实用性**：是否对观众有实际帮助
- **深度**：内容的深度和专业程度
- **创新性**：是否有新的观点或方法

### 适合人群
- **技能水平**：新手/中级/高级
- **领域**：特定专业或广泛受众
- **需求**：学习/工作/娱乐等
- **前置条件**：是否需要相关知识

### 推荐指数
- ⭐⭐⭐⭐⭐ (5星)：内容优质、实用性强、值得反复观看
- ⭐⭐⭐⭐ (4星)：内容较好、有一定价值、推荐观看
- ⭐⭐⭐ (3星)：内容一般、有参考价值、可选择性观看
- ⭐⭐ (2星)：内容较差、价值有限、不推荐
- ⭐ (1星)：内容很差、浪费时间、强烈不推荐

### 观看建议
- **预备知识**：是否需要提前学习某些内容
- **观看方式**：一次性看完、分多次观看、边看边练等
- **重点提示**：哪些部分值得重点关注
- **时间投入**：实际观看需要的时间（考虑暂停、回放等）
